# GF Framework

**Godot 4 轻量级架构框架 —— 关注点分离 · 生命周期管理 · 事件驱动**

## 🌟 核心理念

GF Framework 旨在解决 Godot 项目在中后期常遇到的"逻辑与表现交织"、"强耦合"以及"节点生命周期失控"等痛点。它提供了一套以运行时容器分层为核心，并以 Foundation 纯算法层补足大数、定点数、进度曲线与离线结算能力的清晰架构。

- **逻辑与表现分离**：核心业务逻辑在 `System` 层纯代码运算，远离由场景树决定的时序问题。
- **可控的生命周期**：所有的核心数据(`Model`)和系统(`System`)都由框架管理的 IoC 容器执行严格的三阶段初始化(`Init` -> `AsyncInit` -> `Ready`)。
- **解耦的事件模型**：提供低频强类型的载体事件与高频轻量的字符串事件，满足不同性能敏感度的模块通信需求。
- **清晰的工具边界**：需要注册、持有运行时状态的能力放进 `Utility`；纯值对象、纯算法、纯格式化、纯曲线结算能力放进 `Foundation`，避免所有通用代码都堆到同一层。
- **强大的扩展与工具**：内置分层状态机、动作队列、可撤销命令栈以及各种优化与隔离工具。
